The Reality of Development üõ†Ô∏è

When a professional developer sets out to build something, the first step is always analysis, not coding. They need to figure out the architectural plan before writing the first line of code.

This process starts by defining two key elements:

  1. What needs to be built? (The features)
  2. What tools are required? (The tech stack)

Then comes the essential back-and-forth process that defines real development:

For example, when I build an app:

  • I started with the front-end (React).
  • I coded each component individually.
  • I connected each component to the back-end API.
  • Then, I tested it. When an issue arose, I might have to modify the front-end slightly. If that didn't fit, I would realize I needed to change the back-end API's response structure.

The Iterative Loop üîÑ

Real coding is a continuous, cyclical process‚Äîa loop of trying, failing, and adjusting‚Äîthat moves you closer to the desired outcome:

  • You take a feature or a problem.
  • You figure out the steps required to solve it.
  • You create a code structure.
  • You test it.
  • You find errors (bugs).
  • You fix the errors.
  • You test again.

Slowly but surely, that initial basic function or piece of code starts working.

The most important takeaway? You only need to know what output you want . The " how " is something you figure out as you code, through this iterative, back-and-forth process.
